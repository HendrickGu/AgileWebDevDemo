Concept of Hyperlinks

Definition

Hyperlinks are a fundamental feature of the web, enabling users to navigate between different resources by clicking on links.

Hyperlinks are implemented using the <a> tag in HTML, which defines a hyperlink. When a user clicks on a hyperlink, the browser sends a request to the server specified in the hyperlink's href attribute and retrieves the linked resource.

From a Web Developer's Perspective:

Hyperlinks are created using the <a> tag in HTML. Developers specify the target resource's URL in the href attribute to define the hyperlink's target. For example, <a href="https://www.example.com">Click here</a>

Click here

When a user clicks on a hyperlink, the browser sends an HTTP request to the target URL. This request includes necessary information such as the request type, headers, etc., and the browser waits for the server's response.

From a Browser's Perspective:

When parsing an HTML document, the browser encounters an <a> tag and creates a clickable hyperlink based on the href attribute's value. When a user clicks on this hyperlink, the browser sends an HTTP request to the target URL specified in the href attribute. This request includes metadata like the Referer header, indicating the request's origin page.

After receiving the server's response, the browser processes it accordingly. If the response is an HTML page, the browser parses and renders the page. If the response is a file like an image or video, the browser handles it appropriately.

If the response is an HTML page that contains other hyperlinks, the browser repeats the above steps to load and render other parts of the page until the page is fully loaded.

For example, the following code creates a hyperlink to Google:

<a href="https://www.google.com">Google</a>

When a user clicks on the link, the browser sends a request to https://www.google.com and displays the Google homepage.

Types of Links[1]

Inline Links

Inline links display remote content without embedding it, allowing users to access it directly or by following the link. They can show modified versions of the content, such as thumbnails or previews, instead of the full content. This approach reduces file sizes and improves response times, especially when the full content is not immediately needed.

Example: Click here to visit Example.com.

Anchor Links

Anchor links, also known as fragment links, are used to link to a specific part of a document.

Example: Back to Top

This can be a portion of text, a heading, or even a hot area in an image. Anchors are marked with identifiers, and a link to an anchor is called an anchor link. For example, in HTML, you can create an anchor like this: <a id="name"></a>. To link to this anchor, you would use <a href="#name">Link Text</a>.

  • Text hyperlink: Embedded into a word or a phrase.
  • Image hyperlink: Embedded into an image.
  • Bookmark hyperlink: Takes visitors to another part of a web page.
  • E-mail hyperlink: Allows visitors to send an e-mail message.

Fat Links

Fat links are hyperlinks that contain additional information or metadata. They are often used in semantic web applications to provide context and meaning to the linked resource.

Why Hyperlinks Are Important

All the above mentioned links, complemented by quality content, are important in improving a website’s SEO[2], particularly when they’re done organically or naturally.

To rank in search engines like Google, a website should be properly built using relevant SEO techniques. Another factor is that it should come across as trustworthy and as an authoritative source.

This can be achieved through link building or the process of having other websites link back to your website, which gives you a backlink. As previously mentioned, a site’s backlink profile is an essential ranking factor for Google, along with relevant and quality content.

Hyperlinks are advantageous if used correctly. If implemented otherwise, they can potentially hurt a website’s rankings.

When dealing with hyperlinks, it’s important to avoid artificial inbound links from spammy websites, as these can affect a website’s scores[3]. Additionally, external links to your content should point to high-quality and relevant sites to benefit both you and your readers. The number of links also plays a part in rankings. Having too many outbound links can put a website at risk of looking spammy.

Aside from improving SEO, links can increase a website’s traffic and credibility[4]. This helps in building connections within a particular web community, which then paves the way for link building. Links are also useful when used in conjunction with calls to action, as these can increase conversion rates.

AJAX (Asynchronous JavaScript and XML)[5]

AJAX is a technique used in web development to create interactive web applications. It allows web pages to be updated asynchronously by exchanging small amounts of data with the server behind the scenes. This means that web pages can update without reloading the entire page.

History of Hyperlinks

Please click on the following years to obtain different information for each year

1945: Vannevar Bush proposed the concept of a hypothetical machine called the "memex" in his influential essay "As We May Think[1]," published in The Atlantic Monthly. The memex, short for "memory extender," was envisioned as a mechanical device that would augment human memory and facilitate the organization and retrieval of information.

The telegram was a breakthrough in communication technology, which Vannevar Bush imagined could evolve in unprecedented ways. (Associated Press)

Bush's idea was groundbreaking for its time, as it predated the development of modern computers and the internet. The memex was conceived as a desk-like device with screens for reading microfilm, buttons for navigation, and a keyboard for entering annotations. Users would be able to create, store, and access a personal library of documents and information, similar to the way we use computers today.

One of the key features of the memex was its ability to create links, or "associative trails," between different documents. These links would allow users to navigate between related pieces of information, forming a non-linear network of knowledge. This concept foreshadowed the hypertext and hyperlinks that would later become central to the World Wide Web.

1960s: Ted Nelson made significant contributions to the field of computing by introducing the terms "hypertext" and "hyperlink." He envisioned a revolutionary concept of non-sequential writing, where information could be interconnected in a web-like structure. This groundbreaking idea laid the foundation for modern hypertext systems and the World Wide Web.

Nelson's vision culminated in the proposal of the Xanadu project, an ambitious endeavor aimed at creating a global hypertext system[2]. The Xanadu project sought to enable users to access and navigate a vast network of interconnected documents, allowing for seamless information retrieval and sharing.

Although the Xanadu project faced numerous challenges and was never fully realized in its original form, Nelson's ideas and contributions were instrumental in shaping the development of hypertext and hypermedia systems. His work laid the groundwork for future innovations in information technology and inspired generations of researchers and developers to explore the possibilities of interconnected digital information.

1970s: The Hypertext Editing System (HES) is developed by Andries van Dam and his colleagues at Brown University.

HES allows users to create and edit hypertext documents, laying the foundation for future hypertext systems.

1980s: Tim Berners-Lee works at CERN and develops ENQUIRE, a system for storing and sharing information. This project later inspires the creation of the World Wide Web.

1989: Tim Berners-Lee writes a proposal for the World Wide Web, outlining the basic concepts of hypertext and the HTTP protocol. He also creates the first web browser and web server.

1990: The first web page is created by Tim Berners-Lee, providing an introduction to the World Wide Web and instructions on how to create web pages.

1991: The first website goes live at CERN, providing information about the World Wide Web project and how to use it.

1993: The Mosaic web browser is released, making the World Wide Web accessible to a wider audience. Mosaic introduces features such as inline images and the ability to display text and images on the same page.

1994: The World Wide Web Consortium (W3C) is founded by Tim Berners-Lee to standardize web technologies and ensure their interoperability.

Late 1990s: The dot-com boom sees a rapid increase in the number of websites and internet users, leading to the popularization of hyperlinks as a means of navigating the web.

2000s: Web 2.0 emerges, bringing a new wave of interactive and collaborative web applications. Social media platforms like Facebook and Twitter use hyperlinks extensively to connect users and share information.

2010s: The rise of mobile internet and smartphones leads to a shift towards mobile-friendly web design, with hyperlinks remaining a crucial element of navigation on mobile devices.

Present: Hyperlinks continue to be a fundamental feature of the web, enabling users to navigate between different resources and access information quickly and efficiently. Advances in web technologies have led to the development of new types of hyperlinks, such as deep links and app links, further enhancing the user experience.
